FAQ

{% call faq('Why did you do this?') %}

Because data is delicious.

It started with the Pokéstats project, which quickly showed itself to be very popular on Reddit, including the /r/clopclop subreddit. A user there volunteered to curate the list of ponies and tags, and so this version was born.

I'm calling the site a beta version for now because unlike with Pokémon, there's no hard and fast definitive canon to work with. It needs help to work out the little problems.

{% endcall %} {% call faq('How did you do this?') %}

It's all done in Python.
A little script fetches all the data from the sites, results are parsed out, and a page is generated. The source is even available on Github for your perusal.

The pony database used to generate this site is kept as a simple spreadsheet. A lot of effort has gone into curating that database - it's a lot trickier than it first appears! And there's always room for improvement.

Absolutely!
If you have suggestions about the ponies named or tags used, look at the pony database link above and make suggestions via the contact info at the end of this page.

The tagging is miserable on some of these posts. And without good tags, it's hard to get good data. If you can improve them, that'd be the best, both for me and for the users of the boards themselves.
If you'd rather help with code, the source is on Github, and the most complicated function - by far - is tag analysis. Got ideas? Bring them up there or on Twitter! I'd be happy to take pull requests and discussions.

{% endcall %} {% call faq('Why did you choose these sites?') %}

They're the biggest sites I could easily query. They make it possible to programatically fetch all the post information for a given tag, and from there I can eliminate duplicates (by cross-referencing the md5) and learn more from the sites together than any one provides alone.

e621.net is a general-furry pornography board. It's big and has high tagging standards, making it a key player in this game.

wildcritters.ws supposedly specialize in cute and/or feral material. They don't have much MLP content, but I was already crawling them in the Pokestats project.

rule34.paheal.net is probably the most famous of the rule34 sites. It's got huge volumes of content, but has sadly chosen to not tag sexuality information, so it doesn't help the charts.

rule34.xxx is the largest site on the booru.org network. It's got a lot of porn. Which is great for counting data, but the site's tagging is extremely poor.

ponibooru.org is a specialized boardfor MLP-related material. Unfortunately, they're painfully, painfully slow. Crawling them was a nightmare.

Both of the rule34 sites do not make any effort to seperate MLP character tags from the tags for other characters with the same name. When this happens, it leads to images being included that shouldn't be, so when I see a namespace collision, I exclude the tag from those sites.

If you have any other suggestions, let me know! There's contact info at the end of the page.

{% endcall %} {% call faq("Why don't you scrape DA / FA / Pixiv / etc?") %}

I want to!
But it just can't be done automatically. On most of the sites that function as "primary sources", people don't have much incentive to thoroughly and consistently tag images. And without tagging, there'd be no way to identify porn that belongs here.

On top of that, the sites being used all identify image files the same way - by md5 hash. This makes it possible to tell when multiple sites are talking about the same image without having to download it first. Having to download everything would be incredibly expensive!

{% endcall %} {% call faq("What's with the sexuality charts?", id="sexuality") %}

The idea was just too great to pass up.
They're an attempt to categorize posts based on the tagging provided.
Each post is, if possible, categorized into one of the following buckets:

G: Bisexual/Group. These posts probably contain more than two characters, and that's as much as can be automatically determined.

?: Ambiguous/Intersex/Herm. There's nonbinary gender going on here.

G: Gay.

M: Male (solo).

F: Female (solo).

L: Lesbian.

m: Male (inferred). The image did not have sufficient tagging to put it in any of the above categories, but had tags (ie: "penis") implying it depicts a male.

f: Female (inferred). See above.

s: Straight (inferred). The image was not directly tagged into a category, but picked up signs of both male-inferred and female-inferred. Best guess is, it's straight. Alternatively, the image was directly tagged with 'male' and 'female', but nothing to say it's 'straight' or 'group' or 'ambiguous'.

{% endcall %} {% call faq("Your numbers aren't right!") %}

The counts often don't quite match what shows up when you click the links. There's a couple reasons for this:

  • The searches on e621.net and wildcritters.ws use the -rating:safe filter, to exclude non-porn posts. On their site, the total counts seem to be both heavily cached and not always for exactly what the search query was. Trust the number of pages more than the sidebar.
  • For many of these entries, I crawl multiple tags and combine the results. On e621.net and wilcritters.ws, I can create a link to "any of these tags", but on the other sites that's not available, so the link only goes to the tag that produced the most results.
  • Ponibooru does not display rating:explicit images unless you login or set your user preferences, so be wary of that. The crawler only looks at the "questionable" and "explicit" categories.
If you find another source of error, let me know! I'd be glad to fix it.
